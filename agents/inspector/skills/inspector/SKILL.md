# 出题与总结报告撰写标准

## 报告撰写指南

当调用了 `generate_report.py` 后，你会获得一份 JSON 格式的历史周期薄弱点分析数据。
请遵循以下排版框架生成 Markdown 内容（用于让 export_word 装配成 docx）：

```markdown
# [学生姓名] 的本周学情督导报告

## 总体完成情况
【总结该生学习态度（基于总批次）。比如："本周共完成并批改了 X 次作业，平均得分 XX 分"等赞美之词。】

## 各学科表现概况
【分别说明各科目情况分析。如：数学练习频次高，平均分优良，主要集中在计算错误上...】

## 核心易错点（重点攻克方向）
【提取出现频率最高的 Top3 的错误类型并罗列，点出这几点对后续大考的影响。】

## 改进建议
1. 【建议1】
2. 【建议2】
...不要超过 3 条
```

## 错题本整理排版规范
如果是错题本类型，请严格采用分级罗列：

```markdown
# [学生姓名] 专属错题整理与解析

## [学科名1]
### 错题 1: [日期]
- **题目**：[如果存在题目则写，否则写"见作业底稿"]
- **原答**：[错误答案]
- **正确**：[正解]
- **错因**：[错误类型 - 细化原因]
- **督导拆解思路**：[Thinking_guide中的讲解提示]
...
```

## 根据薄弱点自动出题规范
如果任务是【出题】：
1. 根据上游提供的【核心薄弱点/易错点】，出具相应梯度的典型题目（数量严格匹配命令要求 `count = N`）。
2. 在所有题目之后，**必须通过分页符（Markdown中可不体现，但必须另起一级大标题 `# 答案与解析`）** 给出详细的推导答案，以便家长直接校验。
3. 出题必须符合对应年段的知识大纲（如三年级不要出方程题）。
